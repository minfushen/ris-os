# 首页信息架构改版建议清单

> 基于当前「风控 OS」首页形态（Hero → 总览 → 快捷入口 → 异动探照灯 → 工作项表）及行业内常见的 **态势 → 异常/待办 → 处置 → 闭环** 逻辑整理，可作 PRD 附录或迭代 backlog。

---

## 一、改版目标（建议先写死，避免发散）

1. **首屏 10 秒内**：用户能回答三件事——「我现在有没有必须马上处理的事」「最危险/最紧急的是什么」「点哪里开始干活」。
2. **首页承担「指挥台」**：聚合与跳转；深度配置、报表、规则编辑以二级页为主，不在首页堆全功能。
3. **同一业务对象可贯穿**：从一条异动或任务能带到「客户/申请/策略版本」等上下文（至少 ID + 名称），避免到子模块再搜一遍。

---

## 二、信息架构：建议的纵向层级（自上而下）

| 层级 | 建议承载内容 | 与现状关系 |
|------|----------------|------------|
| L0 全局 | 环境、角色、全局搜索、通知 | 顶栏已有雏形，可强化「与我相关」通知 |
| L1 态势 | 今日/本周关键 KPI、趋势或对比 | 对应「工作台总览」，建议区分「结果指标」与「过程/队列指标」 |
| L2 行动入口 | 高频动作（发起归因、回测、发布、抽检等） | 对应「快捷入口」，建议与 L3 队列联动（从队列反查快捷能力） |
| L3 异常与待办 | 需处置队列：超时、高危、待复核、待发布 | 对应「探照灯 + 任务表」，建议合并为一条「工作队列」心智，或明确分工（探照灯=实时信号，表=工单） |
| L4 明细与下钻 | 抽屉/侧栏/详情页 | 任务抽屉已有，需统一字段与下一步动作 |

---

## 三、分区块改版清单（按首页区块）

### 1. Hero 区

| # | 建议项 | 说明 | 优先级 |
|---|--------|------|--------|
| H1 | 增加「我的待办」数量或最紧急一条摘要 | 与总览区分：Hero 偏「个人/当班」，总览偏「大盘」 | P0 |
| H2 | 主按钮与队列一致 | 主 CTA 对应队列里最高优类型（如今日全是归因类则强化归因） | P1 |
| H3 | 值班/环境信息可折叠 | 常看用户可收起到次要行，释放纵向空间 | P2 |

### 2. 工作台总览（指标卡）

| # | 建议项 | 说明 | 优先级 |
|---|--------|------|--------|
| M1 | 指标分两类：**队列类**（待处理、待复核、超时）与 **风险类**（告警数、通过率变动） | 避免全是「数字卡片」语义重复 | P0 |
| M2 | 每张卡可点击下钻 | 点击跳到预筛选后的任务列表或对应子路由 | P0 |
| M3 | 数据来源标注 | 「实时接口 / T+1 / 演示」避免误信演示数 | P1 |
| M4 | 支持时间范围 | 今日 / 近 7 日 / 本月，与风控周会节奏对齐 | P2 |

### 3. 快捷入口

| # | 建议项 | 说明 | 优先级 |
|---|--------|------|--------|
| Q1 | 与角色绑定（千人千面） | 策略岗突出回测/发布；质检突出抽检；管理层弱化配置项 | P1 |
| Q2 | 区分「发起」与「进入模块」 | 减少误点：有的入口应是「去策略列表」而非直接打开发起弹窗 | P1 |
| Q3 | 最近使用 / 收藏 | 缩短路径，符合企业门户习惯 | P2 |

### 4. 异动探照灯

| # | 建议项 | 说明 | 优先级 |
|---|--------|------|--------|
| S1 | 每条固定 **主行动 + 次行动** | 主：发起归因/捞回/建工单；次：详情/图谱/忽略 | P0 |
| S2 | 分级与 SLA 外显 | 高/中/低 + 剩余处理时间或「已持续 X 分钟」 | P0 |
| S3 | 状态：未读/已认领/已关闭 | 避免列表无限变长；支持「稍后处理」进队列 | P1 |
| S4 | 与任务表联动 | 点「发起」生成任务后，在表中置顶并高亮 | P1 |

### 5. 工作项中心（表格/列表）

| # | 建议项 | 说明 | 优先级 |
|---|--------|------|--------|
| T1 | 默认视图：**我的待办** / **全部** Tab | 队列心智优先于「全表浏览」 | P0 |
| T2 | 列字段收敛为业务语言 | 任务 ID、标题、状态、优先级/SLA、类型、对象、更新人/时间 | P0 |
| T3 | 行内操作不超过 2 个 | 其余收进「更多」；主按钮为「处理」进抽屉或详情 | P1 |
| T4 | 筛选与 URL 或本地状态同步 | 便于分享链接、刷新不丢筛选 | P2 |
| T5 | 空状态分两种 | 无后端：引导启动 API + 演示数据说明；有后端无数据：引导创建首任务 | P0 |

### 6. 跨区块一致性与导航

| # | 建议项 | 说明 | 优先级 |
|---|--------|------|--------|
| C1 | 统一「工作项」模型 | 探照灯项、表行、抽屉用同一套状态枚举与字段名 | P0 |
| C2 | 面包屑与首页关系 | 从子模块「返回工作台」回到带筛选的同一队列视图 | P1 |
| C3 | 错误与降级策略 | 接口失败时明确区分「演示数据」与「真实数据」 | P0 |

## 四、交互与操作逻辑（跨首页）

| # | 建议项 | 说明 | 优先级 |
|---|--------|------|--------|
| I1 | 单条闭环：认领 → 处理 → 提交结论 → 可选复核 | 与策略发布、抽检流程对齐 | P1 |
| I2 | 批量操作（后续） | 批量认领、批量导出、批量加白（需权限） | P2 |
| I3 | 键盘与密度 | 专家用户：表格密度、快捷键（如 `R` 刷新、`/` 聚焦筛选） | P3 |

---

## 五、内容与数据策略

| # | 建议项 | 说明 | 优先级 |
|---|--------|------|--------|
| D1 | 演示数据与真实数据 UI 区分 | 标签、角标、文案「演示环境」 | P0 |
| D2 | 首页加载顺序 | 先 L3 队列与 L1 核心 KPI，再次要模块，避免首屏空白 | P1 |
| D3 | 埋点预留 | 区块曝光、主按钮点击、任务完成漏斗 | P2 |

---

## 六、建议迭代顺序（便于排期）

1. **第一波（P0）**：队列心智（我的待办 + 默认 Tab）+ 探照灯主/次行动 + 总览卡分类与可点击 + 空/错状态 + 演示/真实区分。
2. **第二波（P1）**：角色化快捷入口、探照灯与任务表联动、行内操作收敛、面包屑回带上下文。
3. **第三波（P2/P3）**：时间范围、收藏最近、URL 同步筛选、批量与键盘。

---

## 七、验收标准（一句话）

- 新用户 **30 秒内** 能完成一次「从看到异常 → 点主行动 → 在任务表看到新建项 → 打开处理」的路径。
- 老用户 **不开子模块** 能判断「今天要不要加班救火」。
- 接口挂了仍能看懂 **哪些是演示、哪些该去修环境**。
