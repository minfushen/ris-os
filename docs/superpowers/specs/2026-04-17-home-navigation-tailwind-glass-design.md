# 新首页 / 全局导航骨架视觉设计方案

日期：2026-04-17

状态：已确认的视觉设计方案，待进入实施计划

适用范围：`frontend/` 新首页、全局导航骨架、一级模块页公共壳层

---

## 1. 背景与目标

项目已完成重大产品方向调整，旧的“任务流 / 分析详情 / 信审详情”不再作为主产品路径。新的产品方向是一个面向风控管理、监控分析、策略管控与风险核查的操作系统型前端。

本设计文档只解决一个问题：

把当前项目的前端主界面，统一为一套可持续扩展的 `Tailwind CSS + 浅色磨砂玻璃` 高级风格视觉系统，并为“新首页 / 全局导航骨架”提供明确设计依据。

本设计的核心目标：

1. 让产品第一眼就脱离旧任务系统气质
2. 建立可持续复用的视觉语言，而不是只做一个漂亮首页
3. 用 `Tailwind CSS` 作为主样式体系承接新设计
4. 在不过度扩大改造范围的前提下，优先拿下“看得见的高级感”

---

## 2. 设计结论

本次前端视觉方向正式定为：

`浅色金融玻璃台`

这不是纯白后台风，也不是重霓虹科技风，而是一套面向专业系统的浅色、高级、克制、带材质层次的 UI 表达。

### 2.1 风格关键词

1. 雾白
2. 冷青灰
3. 半透明磨砂玻璃
4. 细描边
5. 柔和高光
6. 克制层次
7. 专业金融终端感

### 2.2 体验目标

用户打开系统后，应当感受到：

1. 这是一个高价值的专业风控系统，不是普通管理后台
2. 页面结构有秩序、有层次、有判断优先级
3. 界面不靠夸张颜色取胜，而靠材质、留白、层级和精度取胜
4. 首页是“操作台”，不是“任务列表页”

---

## 3. 配色体系

### 3.1 基础背景色

页面不使用纯白，而使用带轻微冷感的浅雾背景。

- 页面主背景：`#f6f8fb`
- 背景渐变起点：`#eef4f3`
- 背景渐变终点：`#f8fafc`
- 一级浮层底色：`rgba(255,255,255,0.62)`
- 二级浮层底色：`rgba(255,255,255,0.74)`

设计要求：

1. 页面底层始终要带一点空气感，不允许大片毫无层次的纯白
2. 所有玻璃卡片必须与背景保持可区分的亮度差
3. 浮层不能做成“透明到发灰”，要保留高级材质感

### 3.2 主品牌色

主色不走传统深蓝，而采用更现代、更贵气的冷青灰体系。

- 主色：`#6f8f95`
- 深主色：`#4f6970`
- 高光辅助：`#a8c0c3`

使用规则：

1. 主色用于关键操作、当前激活态、局部结构强调
2. 深主色用于导航选中、标题强调、主按钮 hover
3. 高光辅助仅用于边缘反射、渐变或浅强调，不作为纯文字主色

### 3.3 功能强调色

- 成功：`#5f9b7a`
- 警示：`#d7a85f`
- 风险：`#c77b78`

规则：

1. 只用于状态、风险等级、关键指标方向提示
2. 不允许把成功 / 警示 / 风险色做成大面积背景块
3. 同一屏内的强调色总量要克制

### 3.4 文本颜色

- 主标题：`#1f2a30`
- 正文：`#44525a`
- 次级文案：`#6e7c84`
- 弱提示：`#95a2a9`

规则：

1. 整体文本不允许回到纯黑后台风
2. 层级通过深浅和字重共同建立
3. 次级文案必须明显弱于主文案，但保证可读

### 3.5 边框与阴影

- 亮边：`rgba(255,255,255,0.55)`
- 结构边：`rgba(130,150,160,0.18)`
- 阴影：大范围、低浓度、柔和扩散

规则：

1. 边框比传统后台更轻、更精细
2. 阴影服务于浮起层次，而不是制造厚重感
3. 阴影和描边必须同时存在，才能撑起玻璃材质

---

## 4. 材质系统

本方案的高级感主要来自材质，而不是装饰。

### 4.1 玻璃面板

适用区域：

1. 顶部 Hero
2. 左侧导航
3. 顶栏工具区
4. 指标卡
5. 快捷入口卡
6. 抽屉 / 弹层

材质要求：

1. 半透明白底
2. 中等程度 `backdrop-blur`
3. 极细描边
4. 顶部或左上角轻高光
5. 柔和阴影

视觉结果应当是：

1. 轻透
2. 稳定
3. 精致
4. 不塑料

### 4.2 导航选中态

选中态不使用实心高亮条，而采用玻璃胶囊高亮。

要求：

1. 使用浅青灰半透明底
2. 图标和文字略加深
3. 左侧可带一条细高光边
4. hover 与 active 必须可区分

### 4.3 按钮系统

按钮分为三层：

#### 主按钮

用于：

1. 发起工作项
2. 进入关键模块
3. 首页主操作

规则：

1. 用浅青灰实体底
2. 不做重阴影
3. 圆角中等
4. hover 时提升亮度与细微浮起感

#### 次按钮

规则：

1. 玻璃边框按钮
2. 半透明底
3. hover 时玻璃亮度增强

#### 文字按钮

规则：

1. 用于次级导航、表格操作
2. 颜色偏冷灰青
3. 禁止过于“链接化”的默认蓝色样式

### 4.4 输入框与筛选器

输入类控件统一为浅玻璃输入体系。

规则：

1. 浅色磨砂底
2. 细边框
3. 聚焦时带轻微青灰 glow
4. placeholder 较轻
5. 下拉面板和日期面板也应延续玻璃浮层风格

### 4.5 表格

表格是整个系统里最需要克制的部分。

规则：

1. 表头浅雾白，不用重色
2. 行分隔线极轻
3. 行 hover 为轻提亮
4. 操作列使用轻量按钮
5. 表格区整体更偏“清爽金融终端”，而非重网格报表

## 5. 页面结构设计

### 5.1 全局导航骨架

#### 左侧导航

建议结构：

1. 顶部品牌区
2. 一级导航区
3. 辅助分组信息
4. 底部系统状态或版本标识

导航信息架构建议：

1. 工作台
2. 监控与分析
3. 策略管控
4. 风险核查
5. 特征工程
6. 数据资产

品牌区要求：

1. 产品名称清晰
2. 搭配一行极短副说明
3. 不使用旧任务系统文案

#### 顶栏

建议结构：

左侧：

1. 当前页面标题
2. 当前页面副说明或 breadcrumb

右侧：

1. 全局搜索入口
2. 通知入口
3. 用户入口

顶栏要求：

1. 与侧栏风格统一
2. 不使用整块深色实底
3. 高度克制，不应抢夺首页 Hero 的注意力

### 5.2 新首页结构

首页的角色是“工作台”，不是“列表页”。

建议分为五个层级区块。

#### 第一层：Hero 区

作用：

1. 表达系统身份
2. 提示当前工作焦点
3. 提供主操作入口

内容：

1. 系统标题
2. 一句副标题
3. 2-3 个主操作按钮
4. 右侧辅助态势信息

#### 第二层：总览指标区

作用：

1. 给出系统级态势
2. 告诉用户当前先关注什么

内容建议：

1. 今日风险预警
2. 待处理工作项
3. 策略待发布
4. 异常波动数

#### 第三层：探照灯 / 异动发现区

作用：

1. 展现系统主动发现问题的能力
2. 成为首页最具“系统 intelligence”气质的区域

形式建议：

1. 横向卡片流
2. 每卡只保留一个结论和一个动作
3. 用轻玻璃风险卡表达

#### 第四层：工作项中心

作用：

1. 承接当前工作队列
2. 作为行动落点而非旧详情链路入口

结构建议：

1. 区块标题
2. 筛选栏
3. 工作项表格
4. 右上角工作项发起入口

#### 第五层：快捷入口区

作用：

1. 将关键系统动作前置
2. 把首页做成真正的 action deck

内容建议：

1. 发起归因分析
2. 发起回测
3. 新建策略变更
4. 发起抽检
5. 进入欺诈排查

## 6. 组件级设计原则

建议优先建立以下基础视觉组件：

1. `AppShell`
2. `SidebarNav`
3. `TopBar`
4. `PageHero`
5. `SectionShell`
6. `MetricCard`
7. `QuickActionCard`
8. `GlassButton`
9. `GlassInput`
10. `GlassTag`

这些组件的意义不是“做 UI 库”，而是先把最能体现风格的外层视觉统一下来。

原则：

1. 首页与模块页复用同一组材质组件
2. 通用组件先解决 80% 视觉问题
3. 不让各个页面自行发明样式

---

## 7. Tailwind CSS 迁移边界

本设计已经确认采用：

`Tailwind CSS 作为主样式体系 + 关键基础组件去 Ant Design 化`

为防止实施范围失控，边界如下。

### 7.1 本轮由 Tailwind 主导的部分

以下部分应优先改为 Tailwind 驱动：

1. 全局布局壳
2. 顶栏
3. 左侧导航
4. 首页 Hero
5. 首页各 section 容器
6. 首页指标卡
7. 首页快捷入口卡
8. 通用按钮
9. 通用标签
10. 通用输入框外壳
11. 通用页面标题区
12. 一级模块页公共壳层

结论：

本轮优先替换“所有用户一眼能感知风格的外层 UI”。

### 7.2 暂时保留 Ant Design 的部分

以下部分本轮可保留 Ant Design 的行为能力：

1. 复杂表格功能
2. 复杂表单校验
3. Drawer / Modal 行为层
4. Tabs
5. 一部分成熟的数据录入组件

但要求：

1. 可以保留其行为结构
2. 默认视觉必须通过 Tailwind 包装层或覆盖方式弱化
3. 不允许主视觉继续被默认 Ant Design 风格主导

### 7.3 本轮不做的迁移

本轮明确不做：

1. 全项目彻底去 Ant Design
2. 所有表单控件重写
3. 所有数据组件改成 headless 方案
4. 全量 design token 底层重构

原因：

本轮目标是先建立主视觉与系统壳层，不是重写整个前端基础设施。

---

## 8. Tailwind 设计 token 建议

建议先建立一组语义 token，而不是在组件里散写色值。

### 8.1 颜色语义

1. `bg-app`
2. `bg-surface`
3. `bg-glass`
4. `bg-glass-strong`
5. `text-primary`
6. `text-secondary`
7. `text-muted`
8. `border-soft`
9. `border-glass`
10. `shadow-glass`
11. `accent-primary`
12. `accent-success`
13. `accent-warning`
14. `accent-danger`

### 8.2 材质语义

1. `glass-panel`
2. `glass-card`
3. `glass-toolbar`
4. `glass-pill`
5. `glass-input`

目标：

后续无论是首页、导航还是一级模块页，都从同一套 token 和材质类出发。

---

## 9. 实施顺序建议

建议按以下顺序实施：

1. 接入 Tailwind CSS
2. 建立设计 token 与玻璃基础类
3. 改造 `AppLayout / AppHeader / AppSider`
4. 改造首页 Hero、总览区、快捷入口区
5. 收口工作项中心
6. 再统一一级模块页骨架

原因：

先做“壳”和“材质”，整站气质会立即改变，后续再做页面填充和模块扩展时不会反复返工。

---

## 10. 非目标与约束

### 10.1 非目标

本设计文档不包含：

1. 业务流程细节
2. 后端联调设计
3. 权限体系设计
4. 全量组件库重建
5. 复杂数据交互设计

### 10.2 约束

1. 当前项目仍需保留现有 React 业务结构
2. 当前项目允许阶段性保留 Ant Design 的行为组件
3. 首页必须优先体现产品方向变化
4. 视觉改造必须优先服务“系统感”和“专业感”

---

## 11. 最终结论

本次“新首页 / 全局导航骨架”视觉设计正式确认如下：

1. 风格定位：`浅色金融玻璃台`
2. 主样式体系：`Tailwind CSS`
3. 迁移策略：`关键基础组件去 Ant Design 化，复杂行为层阶段性保留`
4. 页面结构：
   - 首页 Hero
   - 总览指标
   - 探照灯
   - 工作项中心
   - 快捷入口
5. 全局导航结构：
   - 玻璃侧栏
   - 玻璃顶栏
   - 统一模块标题语义
6. 实施策略：
   - 先壳层
   - 再首页
   - 再模块页
   - 最后做测试与样式收口

这份文档作为后续 `Tailwind 迁移实施计划`、`新首页骨架开发任务单` 和 `模块页统一改造` 的视觉依据。
